Product requirement doc guideline

Created on 2025‑03‑01 · Last updated 2025‑03‑15 · Owner: Alex

1. Purpose

This guideline describes how to write a clear and reviewable product requirement document (PRD) for the Yuanti Wiki knowledge base. It should be easy to review and easy to implement for the delivery team.

2. Basic sections

  • Background & goals
  • User stories & scenarios
  • Functional description & acceptance criteria
  • Non‑functional requirements (performance, security, etc.)

3. Notes

PRDs in this space should focus on what and why, while detailed solution and implementation belong to separate tech design docs.

4. Example code block
