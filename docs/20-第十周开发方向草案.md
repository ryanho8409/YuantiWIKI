## 1. V1.1 已交付内容（开发记录）

以下项已随 **Release V1.1** 合入代码库并发布。

### 1.1 深色模式与设置

- **前端**：`ThemeContext` + `localStorage`（`yuanti-theme`）+ `html[data-theme="dark"]`；`index.html` 内联脚本减轻首屏主题闪烁。
- **页面**：`/settings` 中「显示设置」— 深色模式开关。
- **样式**：`index.css` 扩展语义变量与深色覆盖；正文 / ProseMirror 深色可读性基础处理。
- **导航**：顶栏齿轮「个人设置」进入 `/settings`（同页含「显示设置」深色开关，不再单独提供「显示模式」菜单项）。
- **文档**：`docs/19-深色模式后续迭代备忘.md`（跟随系统、服务端同步偏好等 **5 条候选**，**未排期**）。

### 1.2 个人资料 / 账号（第一期 + 第二期）

- **第一期（只读）**：用户名、角色文案、注册时间、最近登录时间（依赖登录成功时更新 `lastLoginAt`）。
- **第二期（可编辑）**：显示名称、邮箱；头像为**服务端存储**（`avatarPath` + 上传/删除接口），早期草案中的外链 URL 方案已废弃。
- **后端**：
  - `User` 模型：`lastLoginAt`、`avatarPath`（服务端上传目录，相对 `UPLOAD_DIR`）；`email` 唯一约束（空值可多条）。头像外链 URL 已废弃。
  - `GET /api/v1/auth/me` 返回完整资料字段；`POST /api/v1/auth/login` 成功时更新 `lastLoginAt` 并返回同结构 `user`。
  - `PATCH /api/v1/auth/profile` 更新资料；邮箱冲突返回 `409` + `EMAIL_IN_USE`（符合错误契约中的稳定 `code` 约定）。
- **前端**：`AuthContext` 中 `User` 扩展资料字段；`/settings` 页面标题为「个人设置」，首块为「个人资料」+ 表单；齿轮菜单「个人设置」进入 `/settings`。

### 1.3 数据库

- 迁移：`prisma/migrations/20260325103000_user_profile_fields`（部署环境请执行 `npx prisma migrate deploy`）。
