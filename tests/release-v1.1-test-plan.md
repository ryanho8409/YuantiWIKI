## 4. P0 必过项（Smoke + 关键链路）

### 4.1 鉴权与错误契约（API 侧）

场景 A：未认证访问受保护资源
- 步骤：
  - 未携带 `Authorization` token，直接请求受保护接口（例如 `GET /api/v1/auth/me` 或任意受保护业务接口）
- 预期：
  - HTTP `401`
  - 返回 JSON 至少包含 `{ code: 'UNAUTHORIZED', message: string }`
- 结果：`Pass`

场景 B：无权限访问受保护资源
- 步骤：
  - 用普通用户 token 访问管理员接口（例如管理端相关资源）
- 预期：
  - HTTP `403`
  - 返回 JSON 至少包含 `{ code: 'FORBIDDEN', message: string }`
- 结果：`Pass`

场景 C：资源不存在
- 步骤：
  - 使用不存在的 `spaceId/pageId` 调用页面详情或删除/读取接口
- 预期：
  - HTTP `404`
  - 返回 JSON 至少包含 `{ code: 'NOT_FOUND', message: string }`
- 结果：`Pass`

场景 D：业务冲突（建议至少测一个 409）
- 步骤：
  - 对存在子节点/无法删除的对象执行删除（例如有子页面的父页面）
- 预期：
  - HTTP `409`
  - 返回 JSON 中 `code` 稳定（例如 `HAS_CHILDREN`），`message` 可用于兜底展示
- 结果：`Pass`

### 4.2 登录恢复与会话完整链路（UI 侧）

场景 E：登录后回到 Dashboard（默认落点）
- 步骤：
  - 用 `admin/admin123` 登录
  - 观察落点是否在 Dashboard
  - 刷新页面，确认仍能恢复登录态（`/auth/me` 链路正常）
- 预期：
  - 登录成功后进入 Dashboard
  - 刷新后头像/用户名仍显示
- 结果：`Pass`

场景 F：退出登录
- 步骤：
  - 点击顶栏 `Sign out`
  - 之后尝试访问一个受保护路由（例如某个 `/space/:id` 或 `/settings`）
- 预期：
  - 被重定向到登录页
- 结果：`Pass`

### 4.3 Space/Page 基础链路（关键 UI）

场景 G：进入 Space → 选择页面 → 阅读态可见
- 步骤：
  - 在首页进入一个可见 Space
  - 从页面树选择一个页面
  - 查看阅读态正文区域是否正常渲染
  - 验证阅读态标题下信息区可见（创建/最后编辑/浏览信息口径与 UI 展示一致）
- 预期：
  - 正常加载正文与标题下信息
- 结果：`Pass`

场景 H：编辑 → 保存 → 回到阅读态 → 生成版本（History 入口至少可用）
- 步骤：
  - 切换到编辑态（Edit）
  - 输入一段测试内容
  - 保存（Save）
  - 回到阅读态，确认新内容可见
  - 打开版本历史（History），确认可看到至少 1 条版本记录
- 预期：
  - 保存后回读正确
  - History 侧板/弹窗可打开且可见版本列表
- 结果：`Pass`

场景 I：版本恢复（Restore 至少执行一次）
- 步骤：
  - 在 History 中选择一个非最新版本（或至少选择可用版本条目）
  - 点击 Restore
  - 重新回到阅读态查看内容是否回到目标版本
- 预期：
  - Restore 后内容回滚可见
- 结果：`Pass`

### 4.4 头像与深色模式（Release V1.1 重点 / 无换页刷新）

场景 J：设置页上传头像 → 顶栏与 Dashboard 欢迎区即时刷新（不换页）
- 步骤：
  - 保持在 `/settings` 页面打开
  - 上传新头像（任意有效图片类型）
  - 不刷新路由，观察：
    - 顶栏头像是否立刻变化
    - 再切回首页（如果切路由也可以，但要求“无需浏览器强制刷新”）
    - Dashboard 欢迎区头像是否立刻变化
  - 删除头像（恢复默认），再观察上述两个位置是否再次变化
- 预期：
  - 顶栏与 Dashboard 欢迎区与设置页预览保持一致
  - 不出现“上传成功但头像仍不变”的缓存错觉
- 结果：`Pass`

场景 K：深色模式开关
- 步骤：
  - 在 `/settings` 切换深色模式开关
  - 观察页面主题变更是否生效（主要看 CSS 变量生效：背景/文字对比）
  - 刷新页面，确认主题保持（基于 `localStorage`）
- 预期：
  - 开关切换即时生效
  - 刷新后仍保持选择
- 结果：`Pass`

### 4.5 导入入口行为（防止提示污染正文）

场景 L：导入功能未开放提示显示位置正确
- 步骤：
  - 在 `SpacePage` 打开“新建页面”浮层（或触发创建页面 popover）
  - 点击“导入为 Wiki 页面”
  - 观察提示文案出现位置
- 预期：
  - 提示出现在创建浮层内部（按钮下方），不应出现在中间文章阅读区域的正文容器底部/中间空态处
- 结果：`Pass`
