# 深色模式 — 后续迭代备忘

> **状态**：未排期；作为下一次迭代（或独立小版本）的**候选 backlog**。  
> **当前已实现**（Release V1.0 起）：`localStorage`（`yuanti-theme`）+ 设置页「深色模式」开关 + `html[data-theme="dark"]` 与 CSS 变量；`index.html` 内联脚本减轻首屏闪烁。

以下 5 条来自实现深色主题后的产品/体验建议，**暂不开发**，仅作记录。

---

## 1. 「跟随系统」外观档位

- **含义**：除「浅色 / 深色」外，增加第三档「与系统一致」：跟随操作系统浅色/深色（浏览器 `prefers-color-scheme`）。
- **价值**：整台设备统一亮暗时，Wiki 无需单独切换。
- **实现要点**：主题状态需支持 `light` | `dark` | `system`；`system` 时监听 `matchMedia('(prefers-color-scheme: dark)')` 变化并同步 DOM；需明确与「用户曾手动选过」的优先级（常见：手动优先或提供「恢复跟随系统」）。

---

## 2. 主题偏好同步到账号（服务端）

- **含义**：将主题写入用户配置（数据库字段或用户设置 API），登录后多设备一致。
- **价值**：适合账号体系成熟、用户跨终端使用的场景。
- **实现要点**：后端字段 + 读写接口；前端与本地 `localStorage` 的合并策略（首次登录覆盖、或仅在无本地偏好时拉取）。

---

## 3. 对比度与长时间阅读（深色可读性）

- **含义**：在深色下微调正文/辅助字色、行高；必要时为 `pre`/`code` 区块单独定义背景与边框 token，避免灰阶糊在一起。
- **价值**：护眼与 WCAG 对比度更稳。
- **实现要点**：在现有 CSS 变量上增量扩展，避免硬编码回潮。

---

## 4. 正文内图片/附件在深色下的观感

- **含义**：用户上传的白底截图等在深色背景上可能刺眼；可考虑正文图片轻微边框、圆角、或（可选）浅色卡片衬底等**克制**处理。
- **价值**：减少「深色模式里只有图片一块发亮」的割裂感。
- **注意**：属内容呈现策略，需与品牌与性能平衡，避免全局滤镜影响色彩准确性。

---

## 5. 动效与无障碍：`prefers-reduced-motion`

- **含义**：当系统开启「减少动画」时，关闭或缩短大面积过渡（如侧栏/面板的 `transition`）。
- **价值**：与系统辅助功能一致，减轻眩晕与分心。
- **实现要点**：`@media (prefers-reduced-motion: reduce)` 下覆盖 `transition`/`animation` 时长。
