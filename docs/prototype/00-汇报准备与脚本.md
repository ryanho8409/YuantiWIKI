## 0.2 原型演示主流程脚本

建议按以下顺序演示，每步控制在 1–2 分钟内，总时长约 5–10 分钟。

| 步骤 | 页面/操作 | 讲解话术（可自行修改） |
|------|-----------|------------------------|
| 1 | **登录页** → 输入账号密码 → 进入 | 「元体知识库部署在内网，员工使用统一账号登录，与公司现有认证打通。」 |
| 2 | **空间列表**：展示多个空间卡片 | 「登录后看到自己有权限的知识空间，例如产品文档、技术规范、运维手册，按空间做分类和权限隔离。」 |
| 3 | **进入某一空间** → 三栏主界面 | 「进入空间后是典型的三栏布局：左侧是空间列表和当前空间的页面树，中间是文档内容，右侧是大纲和属性，对标 ONES 等成熟产品的使用习惯。」 |
| 4 | **点击/新建页面** → 文档阅读页 | 「点击左侧页面树中的一篇文档，中间展示正文，支持面包屑和右侧目录，方便在长文档中跳转。」 |
| 5 | **点击「编辑」** → 文档编辑页 | 「支持富文本编辑，标题、列表、表格、代码块、图片等，满足技术文档和规范文档的编写需求；支持自动保存，减少丢失风险。」 |
| 6 | **打开版本历史** | 「每次保存都会留版本，可查看历史、对比差异，必要时回滚到某一版，保证知识可追溯。」 |
| 7 | **搜索**（可选） | 「支持按关键词在空间内或全局搜索，结果展示标题、路径和摘要，便于快速找到文档。」 |
| 8 | **空间权限配置**（可选） | 「管理员可为空间配置读/写/管理权限，敏感空间仅对指定人员开放，满足内网安全与合规要求。」 |

**收尾话术建议**：  
「以上是元体知识库的核心体验，在信息架构和文档能力上对标 ONES Wiki，同时我们聚焦文档与知识沉淀，不做项目/需求等复杂联动，更轻量、更易落地。接下来可以从 XX 方面投入（如：先做 MVP 再扩展）。」

## 0.3 对标 ONES 维度表

用于汇报时说明「本系统与市场优秀产品的对齐与取舍」。

| 维度 | ONES Wiki | 元体知识库 | 说明 |
|------|-----------|------------|------|
| **信息架构** | Team → Space → Page | Space → Page（组织可选） | 对齐：空间+页面树；简化：单组织不显式 Team |
| **核心页面** | 空间列表、三栏、阅读/编辑、版本、搜索、权限 | 同左，全部覆盖 | 对齐 |
| **编辑能力** | 富文本+块、表格、代码、图片、@ 引用 | 富文本+常用块、表格、代码、图片；@ 页面引用 | 对齐 |
| **权限** | 空间级 + 页面级、读/写/管理 | 空间级（读/写/管理）+ 可选页面级覆盖 | 对齐 |
| **搜索** | 全局/空间内全文 | 全局/空间内全文 | 对齐 |
| **版本** | 版本历史、回滚 | 版本历史、回滚 | 对齐 |
| **与项目/需求联动** | 与 ONES 项目、需求、缺陷深度联动 | 不做 | 简化：专注文档与知识 |
| **部署** | 云端 SaaS | 公司内网 | 增强：数据不出内网、自控 |

**汇报时可强调**：在「文档编辑 + 知识沉淀」核心能力上对标 ONES，在权限与版本上满足企业诉求；通过不做项目联动和内网部署，降低复杂度并满足安全与可控需求。
