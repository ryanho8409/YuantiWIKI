| 界面 | 主要元素 | 原型中需体现 |
|------|----------|--------------|
| **登录/工作台入口** | 账号密码、企业 SSO 入口、品牌区 | 登录页：账号/密码、产品名、简洁品牌区 |
| **空间列表** | 空间卡片或列表、图标/封面、名称、描述、成员/权限提示、进入入口 | 空间卡片：名称、描述、图标、进入操作 |

| **文档阅读页** | 面包屑、标题、正文（富文本渲染）、编辑按钮、分享/更多、右侧 TOC | 面包屑、标题、正文、编辑、右侧大纲 |

## 0.5 ONES 布局与交互特征提炼

基于公开信息与常见知识库产品的归纳，供原型设计时统一参考。

### 导航结构

- **顶栏**：产品 Logo、主导航（如工作台/Wiki）、全局搜索入口、用户头像/设置。
- **空间内**：左侧为「空间列表 + 当前空间页面树」，树支持多级、展开/收起、拖拽排序（可选）；当前页在树中高亮。

### 左树交互

- 展开/收起：点击节点前箭头或图标。
- 选中：点击节点名称，中间区域加载该页。
- 右键或「更多」：新建子页、重命名、移动、删除等。
- 新建入口：空间根或某节点旁的「+」或「新建页面」。

### 中间内容区

- **阅读态**：面包屑在上，标题在下，正文占主区域；顶部有「编辑」等操作。
- **编辑态**：工具栏固定或随焦点出现；正文为可编辑区；顶部或底部有保存/取消或自动保存提示。

### 右栏

- 文档大纲（TOC）：根据标题层级生成，点击锚点跳转。
- 页面属性：创建人、更新时间、标签等（可选）。
- 评论（若做）：按段落或全文的评论入口。

### 常用按钮与文案

| 场景 | 常见文案/样式 |
|------|----------------|
| 主操作 | 「新建页面」「编辑」「保存」— 主按钮样式 |
| 次要操作 | 「取消」「更多」— 次要或文字按钮 |
| 危险操作 | 「删除」「回滚」— 需二次确认，可用危险色 |
| 状态提示 | 「已保存」「保存中…」「未保存」— 文字或小图标 |

原型中建议：按钮层级（主/次/危险）、图标+文字统一、悬停与禁用态一致。

## 0.6 本系统差异化/简化点（汇报用一页说明）

以下可直接作为汇报时的「我们与 ONES 的差异」说明页内容。

### 元体知识库 vs ONES Wiki：差异说明

| 维度 | 说明 |
|------|------|
| **定位** | 元体知识库专注「文档编辑 + 知识沉淀」，不包含项目管理、需求、缺陷等模块；ONES 是「项目 + Wiki」一体化平台。 |
| **使用场景** | 更适合：规范文档、技术方案、流程手册、会议纪要等纯知识沉淀；不需要与任务、迭代、看板绑定的团队，可降低学习与采购成本。 |
| **部署与数据** | 部署在公司内网，数据不出内网，权限由管理员统一分配，满足安全与合规要求；ONES 以云端 SaaS 为主。 |
| **权限模型** | 空间级读/写/管理 + 可选页面级覆盖，与 ONES Wiki 能力对齐，满足「部分空间或页面仅少数人可见」的需求。 |
| **信息架构** | 采用 Space → Page 两级（组织可选），与 ONES 的 Space/Page 对齐，不做显式 Team 层级，更适合单公司内网。 |

**一句话总结**：在文档与知识能力上对标 ONES，在边界上做「专注文档、内网部署、结构轻量」的取舍，便于快速落地并控制成本。
