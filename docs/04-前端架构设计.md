# 元体知识库 — React 前端架构设计

## 1. 技术栈约定

- **框架**：React 18+
- **构建**：Vite（推荐）或 Create React App
- **路由**：React Router v6
- **状态**：服务端状态（空间、页面、用户）以 React Query（TanStack Query）为主；全局 UI/用户身份用 Context 或 Zustand 等轻量方案
- **请求**：axios 或 fetch + React Query
- **UI 组件**：Ant Design / MUI / 自研组件库（与设计规范统一）

---

## 2. 路由与页面结构（对标 ONES 布局）

### 2.1 路由规划

| 路径 | 说明 | 权限 |
|------|------|------|
| `/login` | 登录页 | 未登录可访问 |
| `/` | 首页 / 空间列表 | 已登录 |
| `/space/:spaceId` | 空间首页（默认打开该空间下第一个页面或空白） | 空间读 |
| `/space/:spaceId/page/:pageId` | 空间内阅读/编辑某页 | 空间读/写 |
| `/space/:spaceId/new` | 新建页面（可选，或弹窗创建） | 空间写 |
| `/admin/users` | 用户管理（系统管理员） | 系统管理员 |
| `/admin/spaces` | 空间管理（可选，或合并到空间列表的「管理」入口） | 系统管理员 |

### 2.2 整体布局（三栏，对标 ONES）

```
+------------------+------------------------+-------------------+
|  Logo / 首页      |                        |  大纲 / 属性      |
|  空间列表         |    主内容区             |  （可选）         |
|  （可折叠）       |    文档阅读 or 编辑     |  评论（P2）       |
|                  |                        |                   |
+------------------+------------------------+-------------------+
```

- **左栏**：空间列表 + 当前空间的页面树（可拖拽排序、新建、删除）。
- **中栏**：文档阅读模式 or 编辑模式（Tab 或顶部切换）；面包屑、标题、正文。
- **右栏**：当前页大纲（TOC）、页面属性（创建人、更新时间）、后续可扩展评论。

## 3. 核心模块划分

| 模块 | 职责 | 关键组件/目录建议 |
|------|------|---------------------|
| **认证** | 登录、登出、当前用户、路由守卫 | AuthContext、PrivateRoute、LoginPage |
| **空间** | 空间列表、空间详情、权限展示 | SpaceList、SpaceSidebar、useSpaces |
| **页面树** | 树形展示、增删改、拖拽 | PageTree、PageTreeNode、usePages |
| **文档阅读** | 渲染富文本、TOC、面包屑 | DocViewer、Toc、Breadcrumb |
| **文档编辑** | 富文本编辑、自动保存、版本入口 | DocEditor、AutoSave、VersionHistory |
| **搜索** | 全局/空间内搜索、结果列表 | SearchBar、SearchResults、useSearch |
| **管理** | 用户管理、空间权限配置、空间列表维护 | AdminUsers、AdminSpaces、SpacePermissionEditor |

---

## 4. 状态与数据流

- **服务端状态**：空间、页面、版本、用户、权限、搜索结果等，一律通过 **React Query** 拉取与缓存；写操作用 `mutation` 后 `invalidateQueries` 刷新。
- **全局客户端状态**：当前用户（来自 `/auth/me`）、侧栏折叠、主题等，用 Context 或 Zustand。
- **本地 UI 状态**：弹窗开关、表单、选中节点等，组件内 useState 或 useReducer 即可。

避免把服务端数据放到 Redux 等全局 store 中重复维护。

---

## 5. 文档编辑器选型

- **富文本**：TinyMCE、Quill、TipTap 等均可；需支持标题、列表、表格、代码块、图片上传、链接。
- **存储格式**：与后端约定一致（HTML 或 JSON 块）；若用块存储，可考虑 TipTap 或基于 Slate 的编辑器。
- **自动保存**：防抖（如 2s）将 content 提交 `PATCH /spaces/:spaceId/pages/:id`；保存中/保存成功/失败在 UI 上给予反馈。
- **版本**：编辑区顶部或右侧提供「历史版本」入口，打开版本列表并支持「预览」与「回滚」。

---

## 6. 权限在前端的体现

- **路由级**：进入 `/space/:spaceId` 前请求空间详情或权限接口，无读权限则重定向首页或 403 页。
- **组件级**：根据「当前用户在该空间的角色」控制「新建页面」「编辑」「删除」「管理权限」等按钮显隐；真正写操作仍由后端校验返回 403 时前端提示无权限。
- **接口**：所有请求带认证信息（Cookie 或 Authorization）；401 时跳转登录，403 时提示无权限。

---

## 7. 与设计规范的一致性

- 若项目已接入 `.cursor/skills/ui-ux-pro-max`，组件、色彩、排版、交互应与既有设计系统一致；新页面优先使用设计系统中的组件与 Token。

---

## 8. 可访问性与性能

- **可访问性**：关键操作支持键盘；表单有 label；树节点可 focus。
- **性能**：页面树大时考虑虚拟滚动；文档大时富文本分块渲染或懒加载；列表分页。
