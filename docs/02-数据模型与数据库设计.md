# 元体知识库 — 数据模型与数据库设计

## 1. 设计约束与约定

- **数据库**：PostgreSQL
- **命名**：表名、字段名小写 + 下划线；主键统一为 `id`（UUID 或 bigserial）；时间统一 UTC 存储，展示时按时区转换。
- **软删除**：空间、页面采用 `deleted_at`，重要操作可扩展 `audit_log` 表。

---

## 2. 实体关系概览

```
users          — 用户（或对接 LDAP 时仅存映射与角色）
spaces         — 空间
pages          — 页面（树形，parent_id）
page_versions  — 页面版本历史
space_permissions — 空间级权限
page_permissions  — 页面级权限（P1）
attachments    — 附件（图片/文件）
```

---

## 3. 表结构设计

### 3.1 用户与认证

```sql
-- 用户表（若自建账号；若用 LDAP 可仅存 id/username/display_name/role）
CREATE TABLE users (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  username     VARCHAR(128) NOT NULL UNIQUE,
  display_name VARCHAR(256),
  email        VARCHAR(256),
  password_hash VARCHAR(256),        -- 自建时用；LDAP 可为空
  role         VARCHAR(32) NOT NULL DEFAULT 'user',  -- system_admin | user
  created_at   TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at   TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_users_username ON users(username);
```

### 3.2 空间

```sql
CREATE TABLE spaces (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name        VARCHAR(256) NOT NULL,
  description TEXT,
  icon        VARCHAR(64),           -- 图标标识或 URL
  sort_order  INT NOT NULL DEFAULT 0,
  created_by  UUID REFERENCES users(id),
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
  deleted_at  TIMESTAMPTZ
);

CREATE INDEX idx_spaces_deleted_at ON spaces(deleted_at) WHERE deleted_at IS NULL;
CREATE INDEX idx_spaces_sort ON spaces(sort_order);
```

### 3.3 页面（树形）

```sql
CREATE TABLE pages (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  space_id    UUID NOT NULL REFERENCES spaces(id),
  parent_id   UUID REFERENCES pages(id),   -- NULL 表示空间根下
  title       VARCHAR(512) NOT NULL,
  content     TEXT,                         -- 富文本 HTML 或 JSON 块存储
  content_format VARCHAR(16) NOT NULL DEFAULT 'html',  -- html | slate | block
  sort_order  INT NOT NULL DEFAULT 0,
  created_by  UUID REFERENCES users(id),
  updated_by  UUID REFERENCES users(id),
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
  deleted_at  TIMESTAMPTZ
);

CREATE INDEX idx_pages_space_parent ON pages(space_id, parent_id) WHERE deleted_at IS NULL;
CREATE INDEX idx_pages_space_sort ON pages(space_id, sort_order) WHERE deleted_at IS NULL;
-- 全文检索（PostgreSQL）
CREATE INDEX idx_pages_search ON pages USING gin(to_tsvector('simple', title || ' ' || coalesce(content, '')));
```

### 3.4 页面版本历史

```sql
CREATE TABLE page_versions (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  page_id     UUID NOT NULL REFERENCES pages(id),
  title       VARCHAR(512) NOT NULL,
  content     TEXT NOT NULL,
  content_format VARCHAR(16) NOT NULL DEFAULT 'html',
  version_num INT NOT NULL,
  created_by  UUID REFERENCES users(id),
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_page_versions_page ON page_versions(page_id, version_num DESC);
```

### 3.5 空间级权限

```sql
-- 主体类型：user | role（若引入角色表可扩展）
CREATE TABLE space_permissions (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  space_id   UUID NOT NULL REFERENCES spaces(id),
  subject_type VARCHAR(32) NOT NULL,   -- 'user' | 'role'
  subject_id UUID NOT NULL,            -- user.id 或 role.id
  permission VARCHAR(32) NOT NULL,     -- 'admin' | 'write' | 'read'
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE(space_id, subject_type, subject_id)
);

CREATE INDEX idx_space_permissions_space ON space_permissions(space_id);
CREATE INDEX idx_space_permissions_subject ON space_permissions(subject_type, subject_id);
```

### 3.6 页面级权限（P1，可选）

```sql
CREATE TABLE page_permissions (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  page_id    UUID NOT NULL REFERENCES pages(id),
  subject_type VARCHAR(32) NOT NULL,
  subject_id UUID NOT NULL,
  permission VARCHAR(32) NOT NULL,     -- 'write' | 'read'
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE(page_id, subject_type, subject_id)
);

CREATE INDEX idx_page_permissions_page ON page_permissions(page_id);
```

### 3.7 附件

```sql
CREATE TABLE attachments (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  space_id    UUID NOT NULL REFERENCES spaces(id),
  page_id     UUID REFERENCES pages(id),   -- 可为空，表示空间级资源
  file_name   VARCHAR(512) NOT NULL,
  file_path   VARCHAR(1024) NOT NULL,       -- 存储相对路径或对象键
  file_size   BIGINT NOT NULL,
  mime_type   VARCHAR(128) NOT NULL,
  uploaded_by UUID REFERENCES users(id),
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_attachments_page ON attachments(page_id);
CREATE INDEX idx_attachments_space ON attachments(space_id);
```

## 4. 内容存储策略（content）

- **方案 A**：`content` 存 HTML 字符串。实现简单，富文本编辑器（如 TinyMCE、Quill）直接产出。
- **方案 B**：`content` 存 JSON（块结构，类似 Notion/Slate）。便于版本 diff、未来扩展块类型和协同。推荐中期采用。
- 首版可采用 HTML，在 `page_versions` 中同样存一份快照，便于回滚与对比。

---

## 5. 树形页面查询示例

- **某空间下根页面**：  
  `SELECT * FROM pages WHERE space_id = $1 AND parent_id IS NULL AND deleted_at IS NULL ORDER BY sort_order;`
- **某页面下子页面**：  
  `SELECT * FROM pages WHERE parent_id = $1 AND deleted_at IS NULL ORDER BY sort_order;`
- **整棵树**（一次查空间下所有页面，前端组树）：  
  `SELECT * FROM pages WHERE space_id = $1 AND deleted_at IS NULL ORDER BY parent_id NULLS FIRST, sort_order;`

---

## 6. 全文检索

- 使用 PostgreSQL `to_tsvector` + `plainto_tsquery` 或 `websearch_to_tsquery` 即可实现空间内/全局搜索。
- 若后期文档量很大，可考虑 PostgreSQL 全文检索配置（中文需扩展如 zhparser）或接入 Elasticsearch。
