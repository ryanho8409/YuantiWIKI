# 元体知识库 — 产品迭代计划

> **用途**：按**周/迭代**记录产品在 **功能、体验、性能、稳定性、语言与文案** 等方面的优化与演进，便于回顾决策、对齐团队预期。  
> **维护约定**  
> - 每个迭代对应一次「可交付周期」，通常与 [周开发 Todo](./)（如 `13-第七周开发Todo.md`）同步，但本文件侧重 **对外/对产品** 的表述与结项结论。  
> - **新开迭代**：在下方「迭代记录」中新增一节，写清周期、目标、范围、结项。  
> - **与方向草案关系**：详细需求与验收仍以对应周的「方向草案 / Todo」为准；本文件可做摘要。
